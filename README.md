# Implementando Infraestrutura Automatizada com AWS CloudFormation

## Descrição

Este repositório foi desenvolvido como parte do desafio da DIO com o objetivo de documentar os conhecimentos adquiridos sobre o AWS CloudFormation e o conceito de Infraestrutura como Código (Infrastructure as Code - IaC).

Durante o laboratório foi possível compreender como o CloudFormation automatiza a criação e o gerenciamento de recursos na AWS por meio de Stacks, facilitando a padronização dos ambientes e reduzindo a necessidade de configurações manuais.



## Objetivos do desafio

- Compreender o conceito de Infraestrutura como Código (IaC);
- Conhecer o funcionamento do AWS CloudFormation;
- Entender o que são Templates e Stacks;
- Identificar os benefícios da automação de infraestrutura;
- Utilizar o GitHub para documentar o aprendizado.


##  Tecnologias utilizadas

- AWS CloudFormation
- AWS Management Console
- Git
- GitHub




##  Conteúdos estudados

Durante o laboratório foram abordados os seguintes temas:

- Conceitos de Infrastructure as Code (IaC);
- Benefícios do AWS CloudFormation;
- Formatos de Templates (YAML e JSON);
- Criação e gerenciamento de Stacks;
- Atualização e exclusão de recursos utilizando CloudFormation;
- Comparação entre CloudFormation e Terraform.

-

 💡 O que aprendi

Ao realizar este laboratório, compreendi que o AWS CloudFormation permite automatizar a criação e o gerenciamento de recursos na AWS, tornando o processo mais organizado e seguro.

Também aprendi que as Stacks representam um conjunto de recursos criados a partir de um Template e que esse modelo facilita a replicação de ambientes, reduz erros de configuração manual e contribui para o versionamento da infraestrutura.

Outro ponto importante foi entender a diferença entre o CloudFormation e o Terraform. Enquanto o CloudFormation é um serviço nativo da AWS e voltado exclusivamente para esse ambiente, o Terraform permite gerenciar infraestruturas em diferentes provedores de nuvem.

-

##  CloudFormation x Terraform

| AWS CloudFormation | Terraform |
|--------------------|-----------|
| Serviço da AWS | Ferramenta da HashiCorp |
| Exclusivo para AWS | Multicloud |
| Templates em YAML ou JSON | Arquivos em HCL |
| Integração nativa com os serviços AWS | Suporte a diversos provedores |
| Ideal para ambientes totalmente AWS | Indicado para ambientes com múltiplas nuvens |

##  Conclusão

Este laboratório permitiu compreender como a automação da infraestrutura pode tornar o provisionamento de recursos mais eficiente, padronizado e confiável. Além disso, reforçou a importância da Infraestrutura como Código como uma prática essencial na computação em nuvem.
